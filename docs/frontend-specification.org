* 1. 目的と役割

このフロントエンドは、オンラインビンゴゲームのプレイヤー向け UI を担当します。
現時点で担当している責務は次の通りです。

- プレイヤー名入力
- ルーム作成導線
- ルーム参加導線
- ルーム待機画面の表示
- ゲームセッション画面の表示と操作
- リザルト画面の表示
- ルーム状態の復元
- SSE によるルーム状態のリアルタイム反映
- 通知メッセージのトースト表示

* 4. 画面仕様

** 4.1 タイトル画面

表示要素:

- キッカー =Change Your Destiny=
- ゲームタイトル =I Will BINGO=
- =playerName= 入力欄
- =ルームを作成= ボタン
- =ルームに参加= ボタン

補足:

- タイトル画面の主要 UI は画面の上下左右中央に配置する
- プレイヤー名入力欄の placeholder は =プレイヤー名を入力=

*** 4.1.1 ルーム作成導線

=ルームを作成= 押下時の挙動:

- =playerName= が空ならエラートーストを表示
- 空でなければ =POST /rooms= を実行
- ルーム作成成功後、即座には待機画面へ遷移せず作成モーダルを表示

作成モーダルの表示要素:

- ホストプレイヤー名
- ルーム ID
- コピー按钮
- 作成按钮
- 戻るアイコン

モーダル内の挙動:

- =コピー= でルーム ID をクリップボードへコピー
- =作成= でルーム待機画面へ遷移
- 戻るアイコンで =DELETE /rooms/:roomId= を実行し、仮作成ルームを破棄

*** 4.1.2 ルーム参加導線

=ルームに参加= 押下時の挙動:

- =playerName= が空ならエラートーストを表示
- 空でなければ参加モーダルを表示

参加モーダルの表示要素:

- ルーム ID 入力欄
- 参加按钮
- 戻るアイコン

モーダル内の挙動:

- =参加= で =POST /rooms/:roomId/join= を実行
- 参加成功時はルーム待機画面へ遷移
- 戻るアイコンでタイトル画面に戻る

** 4.2 ルーム待機画面

表示要素:

- ルーム ID
- コピー按钮
- 待機中プレイヤー一覧
- =準備OK= 按钮
- ホストのみ =セッションを開始= 按钮
- =ルームを退室する= 按钮

プレイヤー一覧では次を表示します。

- プレイヤー名
- ホストの場合は =/ HOST= 表示
- READY / WAIT バッジ

待機画面の操作仕様:

- まだ ready 前のプレイヤーは =準備OK= を押せる
- この操作は内部的にカード作成と ready 送信を行う
- 全員が ready 後、ホストのみセッション開始可能
- =ルームを退室する= 押下時は確認モーダルを開く
- ホストの確認文言:
  - =ホストプレイヤーが退室すると、参加している他のプレイヤーも退室します。退室してよろしいですか？=
- 非ホストの確認文言:
  - =ホストプレイヤーが参加を締め切った後の再入室はできません。ルームを退室してよろしいですか？=
- ホストは =DELETE /rooms/:roomId= を使って退室し、非ホストは =POST /rooms/:roomId/leave= を使って退室する
- 確認モーダルには横並びの =はい= / =いいえ= ボタンを表示する

** 4.3 ゲームセッション画面

表示要素:

- 画面上部中央の現在の抽選番号
- 抽選番号の外周リングとして表示される共有イベントゲージ
- 画面上部の =current-number-card= 内で表示する抽選アニメーション
- 自分のビンゴカード
- Bingo / Reach カウンタ
- 進行状態に応じて切り替わる下部ボタン

ゲーム開始オーバーレイの仕様:

- セッションが =in_progress= に入った際、そのセッション ID ごとに 1 回だけ全画面の開始オーバーレイを表示する
- オーバーレイ中央には =BINGO START= と =ビンゴスタート= を縦並びで表示する
- 背景は半透明の白系オーバーレイとし、軽いぼかしをかける
- 開始オーバーレイの表示時間は約 1.4 秒で、軽い拡大を伴いながらフェードアウトする
- 表示中は盤面操作と下部ボタン操作を受け付けない
- 開始オーバーレイ終了後に、そのラウンドの通常抽選演出へ進む

抽選アニメーションの仕様:

- セッション進行中に =session.id + round + currentDrawnNumber= の組み合わせが新しくなったとき、=current-number-card= 内で開始する
- 表示する UI は =draw-reel-shell= のみ
- 数字は 1 から 75 の範囲で高速に切り替わり、終盤で減速してサーバーから受信済みの =currentDrawnNumber= に収束する
- 演出時間はおおむね 2 秒台前半
- 確定番号自体は演出前から state に保持されており、演出は表示上のリールアニメーションのみを担当する
- リールは途中インデックスを段階更新するのではなく、生成済みの縦トラック全体を =translateY= と CSS transition で一括遷移させる
- transition の duration はトラック長に応じて可変とし、前半高速・終盤減速の ease-out カーブで収束させる
- 配色は灰色ベースで、アクティブな数字は濃いグレーで表示する
- 表示全体は円形の器の中心に配置される
- =draw-reel-shell= 自体の白い外枠は表示しない
- 新ラウンド開始直後に =phase === waiting_for_player_actions= となっていても、その抽選キーに対応する演出完了前は下部ボタン領域を灰色の =抽選中= 表示のまま維持する

共有イベントゲージの仕様:

- 共有イベントゲージは抽選番号の外周リングとして表示する
- ゲージ行全体は 3 カラムレイアウトとし、中央カラムにイベントゲージ、右カラムに共有イベント中の =MOVE= ボックスを配置する
- 左カラムは現時点では空き領域として保持し、将来の追加要素用に予約する
- ゲージ値は =room.currentSession.eventGauge= と =eventGaugeMax= から描画する
- リング表現は =conic-gradient= による環状ゲージである
- =eventGauge = 0= のときは進捗色を描画せず、未充填リングのみを表示する
- =eventGauge > 0= のときだけ進捗リングを表示する
- リングの開始角は =0deg=
- ゲージ増加時は CSS カスタムプロパティの補間により滑らかにアニメーションする
- 数値ラベルとして =Shared Event Gauge= と =eventGauge / eventGaugeMax= を表示する
- 補足文として =誰かがマスを開くたびにたまり、満タンで次ラウンドにイベントが発火します= を表示する

共有イベントモーダルの仕様:

- ラウンド開始後に番号抽選演出が完了した時点で、=room.currentSession.phase === "waiting_for_event_choices"= なら全プレイヤーにモーダルを表示する
- モーダルを開いた直後は =運命に抗え!= の見出しだけを中央表示する
- 見出し単独表示から 1 秒後に、カード選択 UI 全体へ切り替える
- カード選択 UI の見出しは =運命に抗え!=、説明文は =どちらか一枚を選びとってください=
- 各プレイヤーには自分の担当区間だけが割り当てられる
- 画面中央には、移動量の数字だけを描いた正方形カードを横並びで 2 枚表示する
- カード面に表示する数字は =eventChoice.options[].step= の値である
- =to= や =goalNumber= はイベント開始時点ではクライアントに見せない
- プレイヤーは 2 枚のうち 1 枚を選択し、下部の =運命を変える= ボタンで提出する
- 選択中カードは強調表示する
- 提出前の下部ボタンは赤系の =運命を変える=
- 提出済みの場合、カード選択 UI は無効化され、下部ボタンは灰色系の =他のプレイヤーを待っています= に切り替わる
- 全プレイヤーの提出完了までは通常のカード操作やスキップ進行へ移らない
- モーダルは =waiting_for_event_choices= に入った瞬間には開かず、そのラウンドの抽選演出が完了した後に表示する
- =POST /rooms/:roomId/session/event-choice= 成功時、フロントは通知文言のみ即時更新し、=room= の反映は HTTP レスポンスではなく =room_updated= SSE を待つ
- 全員提出後、クライアントは =resolvedTimeline= を受け取り、区間ごとの =from= / =to= / =selectedStep= / =direction= を使って番号遷移を描画する
- 各区間の番号遷移は =from= と =to= の 2 点だけを見せるのではなく、移動方向に沿った連続番号列を生成し、横スクロールで順に通過させる
- 時計回りなら =1 -> 2 -> ... -> 75 -> 1=、反時計回りなら =75 -> 74 -> ... -> 1 -> 75= の循環規則に従って中間番号を補完する
- 時計回りのときは数列が右から左へ流れ、反時計回りのときは左から右へ流れる
- 横スクロールの duration は移動量に応じて可変とし、短い移動ほどやや長く、終盤で減速して見える ease-out カーブを使う
- 最終的に =goalNumber= がそのラウンドの確定番号として採用され、その番号に対する通常アクションへ戻る

通常抽選リールの途中数字生成:

- 通常抽選のリール演出では、途中数字をランダム値ではなく確定番号へ到達する循環連番として生成する
- 数列は =... -> 73 -> 74 -> 75 -> 1 -> 2 -> 3= のように 75 を越えたら 1 へ戻る規則に従う
- 描画上は生成した全ての途中数字を縦方向に並べ、順番に送ることで最終的に確定番号へ収束させる

同期表示について:

- =syncStatus= 自体は状態として保持
- UI 上の同期項目は現在 =hidden= で非表示
- コード上には「現在は UI 上で使っていないため非表示のまま保持している」旨のコメントを残している

現在 UI に表示していない要素:

- ラウンド数見出し
- フェーズ表示
- 参加プレイヤー一覧
- 抽選履歴
- 上部のアクション説明文

プレイヤーの操作仕様:

- 抽選アニメーション中は全プレイヤー共通でカード操作と =スキップする= を禁止する
- 抽選アニメーション中の進行ボタン領域は、全プレイヤー共通で灰色の =抽選中= ボタン表示に切り替える
- 抽選番号がカード上にある場合は対象マスが強調表示され、そのマスを押す
- このとき下部には灰色の =番号のマスを開いてください= ボタンを表示する
- 抽選番号がカード上にない場合は赤色の =スキップする= ボタンを押す
- 各プレイヤーは 1 ラウンドにつき 1 回だけ操作できる
- 自分の操作完了後、まだ全員完了していない間は灰色の =他プレイヤーの操作待ちです= を表示する

ホストの操作仕様:

- 全員の act 完了後のみ、緑色の =次のラウンドを始める= を押せる

非ホストの表示仕様:

- 全員の act 完了後は、灰色の =次のラウンドが始まります= を表示する

Bingo / Reach カウンタの色:

- Bingo = 0= かつ Reach = 0= のときは灰色
- Reach > 0 かつ Bingo = 0= のときは黄色
- Bingo > 0 のときは緑色

* 6. ストレージ仕様

ブラウザの =sessionStorage= を利用して次の値を保持します。

* 9. SSE 仕様

ルームに入室後、=GET /rooms/:roomId/events= に対して =EventSource= を接続します。

現在の扱い:

- =room_updated= イベントを購読
- 受信した =payload.room= でフロント状態を更新
- 共有イベントの提出後も =room= 同期の正本は =room_updated= とし、HTTP 成功レスポンスではローカルの =room= を更新しない
- =room_closed= イベントを購読
- =room_closed= 受信時はローカルのルーム関連 state と =sessionStorage= をクリアし、タイトル画面へ戻る
- =room_closed= の =message= はトーストとして表示する
- 接続開始時は =connecting=
- 接続済みで =live=
- エラー時は =reconnecting=

ただし、前述の通り =syncStatus= の表示自体は UI 上では非表示です。
