* 1. 目的と役割

このバックエンドは、オンラインビンゴゲームのルーム管理と進行管理を担当します。
現時点で担当している責務は次の通りです。

- ルーム作成
- プレイヤー参加
- プレイヤー退室
- 各プレイヤーのカード作成
- ready 管理
- ホストによるゲーム開始
- 各ラウンドの抽選番号管理
- 各プレイヤーのラウンド行動受付
- ビンゴ判定とゲーム終了判定
- SSE によるルーム状態のリアルタイム配信

* 3. データモデル

** 3.1 Room

ルームは次の情報を持ちます。

- =id=: ルーム ID
- =hostPlayerId=: ホストのプレイヤー ID
- =players=: プレイヤー配列
- =currentSession=: 現在のゲームセッション

** 3.2 Player

- =id=: プレイヤー ID
- =name=: 表示名

プレイヤー ID やルーム ID、セッション ID は =Math.random()= ベースの簡易 ID で生成されます。

** 3.3 GameSession

- =id=: セッション ID
- =status=: =waiting= / =in_progress= / =finished=
- =phase=: 進行段階
- =round=: 現在ラウンド数。開始前は =0=
- =currentDrawnNumber=: 現在ラウンドの抽選番号。開始前は =null=
- =drawnNumbers=: これまでに最終確定した番号一覧
- =eventGauge=: 共有イベントゲージの現在値
- =eventGaugeMax=: 共有イベントゲージの最大値
- =currentEvent=: 共有イベント進行中の一時状態。非イベント時は =null=
- =endReason=: =null= / =bingo= / =all_numbers_drawn=
- =winners=: 勝者となった =playerId= の配列
- =endCondition=: 終了条件
- =playerStates=: プレイヤーごとの状態

デフォルトの終了条件は次です。

- =bingoCount = 1=
- =finishWhenAllNumbersDrawn = true=

** 3.4 PlayerSessionState

- =card=: ビンゴカード。未作成時は =null=
- =isReadyForStart=: 開始準備完了フラグ
- =hasActedThisRound=: そのラウンドで操作済みか
- =hasSubmittedEventChoice=: そのラウンドでイベントカード選択を提出済みか

** 3.5 EventState

共有イベント発火中は =GameSession.currentEvent= に次を保持します。

- =startNumber=: イベント前に通常抽選された番号
- =goalNumber=: イベント後に最終確定番号として採用される番号
- =relayNumbers=: 中継番号の配列。件数は =参加人数 - 1=
- =segments=: プレイヤー担当区間の配列
- =resolvedTimeline=: 全員提出後に確定した区間ごとの結果

** 3.6 EventSegment

- =order=: 実行順
- =assignedPlayerId=: その区間を担当するプレイヤー ID
- =from=: 区間始点番号
- =to=: 区間終点番号
- =clockwiseStep=: 時計回りに進む移動量
- =counterClockwiseStep=: 反時計回りに進む移動量
- =selectedDirection=: プレイヤーが選んだ方向。未提出時は =null=
- =selectedStep=: プレイヤーが選んだ移動量。未提出時は =null=

共有イベントゲージはプレイヤー個別ではなく =GameSession= 側に保持します。

* 4. ビンゴカード仕様

** 4.1 基本構造

カードは =5x5= の 25 マスで構成され、内部的には一次元配列で保持します。

- =positionId=: =0..24=
- FREE マス位置: =12=
- 数字の範囲: =1..75=

各セルは次を持ちます。

- =positionId=
- =value=: 数字。FREE マスは =null=
- =isOpened=: 開いているか
- =isFree=: FREE マスか

** 4.2 生成ルール

カード生成時は =1..75= から重複なしで 24 個をランダム抽出し、FREE 以外の 24 マスへ順に配置します。
一般的な B/I/N/G/O 列ごとの数値帯ルールは採用していません。

カード生成直後の仕様:

- 全セルが =isOpened = false=
- FREE マスも生成直後は閉じている

** 4.3 FREE マスの扱い

FREE マスはカード生成時には開かれません。
=POST /rooms/:roomId/session/ready= を実行した時点で FREE マスが開きます。

** 4.4 抽選番号による開封

各ラウンドでプレイヤーが =act= を行うと、現在の抽選番号と一致するセルがあればそのセルを開きます。
一致する数字がなければカードは変化しませんが、行動済みにはなります。

** 4.5 判定

ビンゴ判定対象のラインは 12 本です。

- 横 5 本
- 縦 5 本
- 斜め 2 本

=bingoCount= は全開しているライン数です。
=reachCount= は「そのライン内で開いているマス数が 4」のライン数です。

* 5. 進行状態

** 5.1 status

- =waiting=: 開始前
- =in_progress=: 進行中
- =finished=: 終了

** 5.2 phase

- =waiting_for_ready=: カード作成と ready を受け付ける
- =waiting_for_host_start=: 全員 ready 後、ホスト開始待ち
- =waiting_for_event_choices=: 抽選後に共有イベントのカード選択待ち
- =waiting_for_player_actions=: 各プレイヤーの act 待ち
- =waiting_for_host_next_round=: 全員 act 後、ホストの次ラウンド待ち
- =finished=: セッション終了

** 5.3 状態遷移

#+begin_example
waiting / waiting_for_ready
  ├─ プレイヤー参加
  ├─ 各プレイヤーが setup
  └─ 全員 ready で waiting_for_host_start へ

waiting / waiting_for_host_start
  └─ ホストが start で in_progress / waiting_for_player_actions へ

in_progress / waiting_for_player_actions
  └─ 全員が act すると
       ├─ 終了条件達成なら finished / finished
       └─ 未達なら waiting_for_host_next_round へ

in_progress / waiting_for_host_next_round
  └─ ホストが next-round で
       ├─ イベント条件未達なら waiting_for_player_actions へ
       └─ イベント条件達成なら waiting_for_event_choices へ

in_progress / waiting_for_event_choices
  └─ 全プレイヤーが event-choice を提出すると
       ├─ currentDrawnNumber に goalNumber を採用
       ├─ drawnNumbers に goalNumber を追加
       ├─ eventGauge を 0 にリセット
       └─ waiting_for_player_actions へ

finished / finished
#+end_example

* 6. ゲーム進行仕様

** 6.1 ルーム作成

ホスト名を受け取り、新規ルームとホストプレイヤーを作成します。
ルーム作成時点で =currentSession= も同時に初期化されます。

初期セッション状態:

- =status = waiting=
- =phase = waiting_for_ready=
- =round = 0=
- =currentDrawnNumber = null=
- =drawnNumbers = []=
- =eventGauge = 0=
- =eventGaugeMax = 0=
- =currentEvent = null=
- =winners = []=
- =endReason = null=

** 6.2 参加

プレイヤー参加は、次の条件を両方満たすときだけ可能です。

- =currentSession.status === "waiting"=
- =currentSession.phase === "waiting_for_ready"=

したがって、全員 ready 後やゲーム開始後は新規参加できません。

** 6.3 setup

プレイヤー自身のカードを 1 回だけ作成できます。

条件:

- ルームが存在すること
- セッションが =waiting= であること
- 対象プレイヤーが存在すること
- そのプレイヤーがまだカード未作成であること

結果:

- ランダムなカードが =playerStates[playerId].card= に保存される

** 6.3.1 共有イベントゲージ

共有イベントゲージは、ゲーム進行中のセッション全体で 1 本だけ管理します。

現時点の実装仕様:

- 保持場所は =GameSession=
- 通常マスを新しく 1 つ開いたときだけ =+10=
- スキップ時は増加しない
- ゲージが最大値以上の状態で次ラウンドが開始されると、そのラウンドはまず共有イベント選択フェーズへ入る
- 全プレイヤーがイベントカード選択を提出した時点で =eventGauge= を =0= にリセットする
- FREE マスの初期開封は加算対象外
- =eventGaugeMax = 30 * セッション開始時の参加人数=
- =eventGaugeMax= はセッション開始時に一度だけ確定し、進行中は再計算しない

補足:

- ゲージ満タン時のイベント生成と選択 API までは実装済みである
- クライアント側のカード UI と番号遷移演出は今後の実装対象である

** 6.4 leave

非ホストプレイヤーは待機中ルームから退室できます。

条件:

- ルームが存在すること
- =playerId= が指定されていること
- 対象プレイヤーが存在すること
- 対象プレイヤーがホストではないこと
- =currentSession.status !== "in_progress"=

結果:

- =players= 配列から対象プレイヤーを削除
- =playerStates= から対象プレイヤーの状態を削除
- セッションが =waiting= 中なら、残存プレイヤーの ready 状態に応じて =phase= を再計算
  - 全員 ready なら =waiting_for_host_start=
  - そうでなければ =waiting_for_ready=
- 更新後のルームを =room_updated= で SSE 配信

補足:

- ホストは =leave= を使えず、既存の =DELETE /rooms/:roomId= で退室する
- ゲーム進行中の退室は =409= で拒否する

** 6.5 ready

ready は開始前かつ =waiting_for_ready= フェーズでのみ受け付けます。

条件:

- プレイヤーが存在すること
- プレイヤー状態が存在すること
- カード作成済みであること
- まだ ready 済みでないこと

結果:

- FREE マスを開く
- =isReadyForStart = true=
- 全員 ready なら =phase = waiting_for_host_start= に遷移

** 6.6 start

ホストのみ実行できます。

条件:

- リクエストの =playerId= が =hostPlayerId= と一致すること
- セッションが =waiting= であること
- フェーズが =waiting_for_host_start= であること
- 全員 ready であること

結果:

- 1 つ目の抽選番号を決定
- =status = in_progress=
- =phase = waiting_for_player_actions=
- =round = 1=
- =currentDrawnNumber= を更新
- =drawnNumbers= に反映
- =eventGauge = 0= に初期化
- =eventGaugeMax = 30 * プレイヤー人数= に初期化
- =currentEvent = null=
- 全プレイヤーの =hasActedThisRound = false=

抽選可能番号が残っていない場合:

- =status = finished=
- =phase = finished=
- =endReason = all_numbers_drawn=
- HTTP ステータスは =409=

** 6.7 act

プレイヤー操作は、進行中かつ =waiting_for_player_actions= の間だけ受け付けます。

条件:

- プレイヤーが存在すること
- カード作成済みであること
- まだそのラウンドで act していないこと
- =currentDrawnNumber= が存在すること

結果:

- 抽選番号と一致するセルがあれば開く
- =hasActedThisRound = true=
- 一致セルが通常マスかつ未開封だった場合は =eventGauge += 10=

全員 act 後の追加処理:

- =winners= を再計算
- 全プレイヤー中の最大 =bingoCount= を計算
- 終了条件を判定

終了条件成立時:

- =status = finished=
- =phase = finished=
- =endReason = bingo= または =all_numbers_drawn=

未成立時:

- =phase = waiting_for_host_next_round=

勝者判定は「=countBingoLines(card) >= endCondition.bingoCount= を満たすプレイヤー全員」です。

** 6.8 next-round

ホストのみ実行できます。

条件:

- リクエストの =playerId= が =hostPlayerId= と一致すること
- セッションが =in_progress= であること
- フェーズが =waiting_for_host_next_round= であること
- 全員が act 済みであること

結果:

- 次の番号を抽選
- =round= を 1 増加
- ゲージ満タンでない場合:
  - =currentDrawnNumber= を抽選番号に更新
  - =drawnNumbers= にその番号を追加
  - =currentEvent = null=
  - =phase = waiting_for_player_actions=
- ゲージ満タンの場合:
  - 抽選番号を =currentEvent.startNumber= として保持
  - 未確定番号から =goalNumber= を 1 つ選ぶ
  - =relayNumbers= を =参加人数 - 1= 個ランダム選出する
  - 区間列を作り、各プレイヤーへ担当区間をランダム割当する
  - =currentDrawnNumber= は開始番号のまま保持する
  - =drawnNumbers= はまだ更新しない
  - =phase = waiting_for_event_choices=
- 全プレイヤーの =hasActedThisRound = false=
- 全プレイヤーの =hasSubmittedEventChoice = false=

抽選可能番号が残っていない場合:

- =status = finished=
- =phase = finished=
- =endReason = all_numbers_drawn=
- HTTP ステータスは =200=

- =playerId=

- HTTP =404=: ルームなし

現在のルーム状態取得。

ホスト退室、および部屋の終了。

リクエスト:

- =playerId=: 必須。ホスト ID

- =message=

失敗:

非ホストの待機中退室。

リクエスト:

- =playerId=: 必須

失敗:

- HTTP =404=: ルームなし
- HTTP =400=: =playerId= 未指定
- HTTP =404=: プレイヤーなし
- HTTP =403=: ホストが =leave= を使用した
- HTTP =409=: ゲーム進行中

** 7.6 POST /rooms/:roomId/session/setup

プレイヤーのカード作成。

リクエスト:

- =playerId=: 必須

- HTTP =200=

失敗:

- HTTP =404=: ルームなし
- HTTP =409=: セッション開始済みまたは終了済み

** 7.7 GET /rooms/:roomId/events

SSE 購読開始。

失敗:

- HTTP =404=: ルームなし

** 8.2 SSE イベント

=room_updated=

- 通常のルーム更新時に使用
- ペイロードは =GET /rooms/:roomId= と同等の =buildRoomResponse(room)=

=room_closed=

- ホスト退室で部屋を閉じるときに使用
- ペイロードは =message= を持つオブジェクト
- 送信後、SSE レスポンスを終了し、ルーム自体を削除する

* 9. バリデーションと制約

- =name= は文字列で、trim 後に空文字だと不正
- =playerId= の型検証は厳密にはしておらず、存在確認で判定している
- 認証や権限制御は =hostPlayerId= 一致確認のみ
- ルーム人数上限は未実装
- セッションの作り直し、再戦は未実装
- 非ホストの途中退室は待機中のみ可能で、進行中は不可
- ホスト退室時は部屋全体を閉じる
- SSE のイベント種別は =room_updated= と =room_closed=

* 10. 実装上の注意点

- =broadcastRoom(room)= はルーム更新時に購読中クライアントへ =room_updated= を配信する
- =closeRoom(roomId, eventName, payload)= は指定イベントを配信してから SSE を閉じ、ルームを削除する
- SSE 購読者がいない場合、ブロードキャストは何もしない
- ビンゴ終了判定は「そのラウンドで全員の act が終わった後」にのみ行う
- =drawnNumbers= は通常抽選番号ではなく「最終確定番号の履歴」である
- =all_numbers_drawn= 判定もこの最終確定番号履歴をもとに行う
